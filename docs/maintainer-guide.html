  3. 🧭 เริ่มอ่านโค้ดจากตรงนี้ (8 สเต็ป)

  6. 📁 แผนผังไฟล์

รันยังไง

npm install
npm run dev      # http://localhost:3000  (Turbopack)
npm run build    # production build = ตรวจ type ทั้งโปรเจ็ค (ใช้แทน test)
npm run start    # รัน production
npm run lint

ต้องมีไฟล์ .env.local (ไม่ขึ้น git) ใส่ ORS API key: 

ORS_API_KEY=<key จาก openrouteservice.org/dev>
ไม่มี key ก็รันได้ — เส้นทางจะ fallback เป็นเส้นตรง (ไม่เกาะถนน)

🧭 เริ่มอ่านโค้ดจากตรงนี้

อ่านตาม 8 สเต็ปนี้แล้วเข้าใจทั้งระบบ แต่ละอันบอกว่า ทำไมสำคัญ และ ดูอะไร

1src/types.ts — รูปร่างข้อมูลทั้งหมด

เริ่มที่นี่เสมอ ดู 3 interface หลัก: Checkpoint (หมุด), RouteDef (เส้นทาง), RunRecord (ผลการวิ่งที่เซฟ) เข้าใจ field พวกนี้แล้วที่เหลือตามง่าย

2src/data/checkpoints.ts — แหล่งความจริงเดียวของหมุด ⭐

CHECKPOINTS คือ array จุดมรดก ๑๐ จุด แต่ละจุดมี coord: [lat,lng], points (แต้ม), fact (เกร็ดความรู้) พิกัด/แต้ม อยู่ที่นี่ที่เดียว — แก้ที่นี่แล้วทุกเส้นทางอัปเดตตาม (แคลอรี่ย้ายไปเป็นค่า "ต่อช่วง" ใน routes.ts — ดูข้อ 3)

3src/data/routes.ts — นิยามเส้นทาง Basic

ดู 2 ฟังก์ชัน:

  • checkpointPath(ids) — แปลง id หมุด → พิกัดเรียงกัน (เส้นทางเริ่มต้น)
  • basic(spec) — factory รับแค่ checkpointIds แล้ว เติม path + distanceKm + legCalories ให้อัตโนมัติ เพิ่มเส้นทางใหม่แค่ก๊อป block ใส่ id หมุด ไม่ต้องพิมพ์พิกัดเอง
  • legCalories: number[] — แคลอรี่ ต่อช่วง (leg) ยาว = จำนวนหมุด − 1 เช่น [90, 70, 130] = ช่วง ๑→๒ ได้ 90, ๒→๓ ได้ 70, ๓→๔ ได้ 130 · กรอกเองหรือเว้นไว้ให้คิดจากระยะ (legCaloriesFromPath)

4src/store/runStore.ts — pushPosition() คือหัวใจ ⭐⭐⭐

ฟังก์ชันสำคัญที่สุดในโปรเจ็ค เรียกทุกครั้งที่ได้ตำแหน่งใหม่ (sim หรือ GPS) ทำ 4 อย่างเรียงกัน:

  1. กรอง noise — ตำแหน่งกระโดด > 200ม. (GPS_JUMP_MAX_M) ทิ้ง
  2. บวกระยะทาง (Haversine)
  3. ต่อ trace (เส้นน้ำเงินที่วิ่งจริง)
  4. เช็คอิน — วน checkpointIds ถ้าเข้าใกล้หมุดที่ index i < 45ม. (CHECKIN_RADIUS_M) และยังไม่เคยเช็คอิน → บวกแต้ม + บวกแคลของช่วงที่เพิ่งวิ่งมา (route.legCalories[i-1], จุดเริ่ม = 0) + เด้ง toast

action อื่น begin/tick/finish/reset ตรงไปตรงมา

5src/app/run/[routeId]/page.tsx — ตัวประสานงาน

หน้าวิ่งจริง รวมทุกอย่างผ่าน useEffect 4 ตัว:

  1. หา route + ดัดเส้นเกาะถนน — โหลด route แล้วเรียก snapToRoads() (โชว์เส้นตรงก่อน → อัปเกรดเมื่อ ORS ตอบ)
  2. ลูปติดตามตำแหน่ง — sim: setInterval ป้อน densify(path) ทุก 180ms / gps: watchPosition + tick 500ms
  3. เซฟตอนจบ — repo.addRun() ครั้งเดียว (กันซ้ำด้วย savedRef)
  4. reset ตอนออกจากหน้า

6snapToRoads.ts + api/route/route.ts — เส้นเกาะถนน (ORS)

  • snapToRoads(points) (client) — POST ไป /api/route, มี cache + fallback เส้นตรง ถ้าล่ม
  • route.ts (server) — proxy เรียก ORS foot-walking, เก็บ API key ฝั่ง server เท่านั้น, แปลงพิกัด [lat,lng]↔[lng,lat]

7lib/geo.ts + lib/stats.ts — คณิต/ค่าคงที่

  • geo.ts: distanceM (Haversine), pathLengthM, densify (แตกจุดถี่ให้ sim ลื่น), metersToKm, legCaloriesFromPath (แคลต่อช่วงคิดจากระยะ), centroid
  • stats.ts: steps() (ประมาณก้าว = ระยะ ÷ 0.75ม.), formatTime, ค่าคงที่เช็คอิน/กรอง GPS
แคลอรี่ ไม่ใช้สูตร MET แล้ว — เป็นค่า ต่อช่วง กำหนดใน legCalories ของแต่ละเส้นทาง (routes.ts) บวกสะสมตอนเช็คอินใน store

8src/lib/routeGen.ts — โหมด Advance

อ่าน top-down: pickCheckpoints() (คัดจุดตามย่าน/มรดก) → orderByNearestNeighbour() (เรียงให้ต่อเนื่อง) → generateAdvanceRoute() ประกอบเป็น RouteDef

(เสริม) components/RunMap.tsx วาดแผนที่ client-only · lib/storage.ts RunRepository ออกแบบให้สลับเป็น Supabase ได้โดยไม่แตะหน้าเว็บ

⚠️ Gotchas (เคยเจอจริง)

  • callback ที่ส่งให้ component ที่ re-render ถี่ ต้องเก็บใน ref ไม่งั้น setTimeout ใน effect ถูกรีเซ็ตทุก frame (toast เคยปิดไม่ได้เพราะเหตุนี้ — ดู CheckinToast.tsx)
  • ใน dev เห็น POST /api/route ยิง 2 ครั้ง/หน้า = React StrictMode เรียก effect ซ้ำ — production ยิงครั้งเดียว
  • Next 16: params เป็น Promise ต้อง unwrap ด้วย use(params)
  • Leaflet ต้องโหลด client-only — RunMap import แบบ dynamic(..., {ssr:false})
  • GPS จริงบนมือถือต้องใช้ HTTPS (localhost ยกเว้น) — เดโมผ่าน LAN IP ใช้โหมดจำลองแทน
