ออกแบบเส้นทางเอง (Advance)

ผู้ใช้เลือก ย่าน + มรดก + บรรยากาศ แล้วระบบร้อยหมุดเป็นเส้นทางวิ่งสด ๆ — เอกสารนี้อธิบายว่ามัน ทำงานยังไงตั้งแต่กดปุ่มจนถึงหน้าวิ่ง

ภาพรวม flow

เส้นทางแบบ Advance ไม่ได้เขียนไว้ล่วงหน้าใน data/routes.ts เหมือนเส้นทาง Basic — แต่ สร้างสด ๆ จากตัวเลือกของผู้ใช้ แล้วฝากไว้ใน localStorage เพื่อส่งต่อไปหน้าวิ่ง 

/build                generateAdvanceRoute()         /run/<id>
เลือก ย่าน/มรดก/  ──▶  • คัดหมุดตามตัวเลือก      ──▶  • หา Basic ก่อน
บรรยากาศ              • เรียงด้วย nearest-neighbour    ไม่เจอ → getDraftRoute()
   │                  • คิดระยะ + แคลอรี่/ช่วง         • snapToRoads (ORS เกาะถนน)
   │                  • คืน RouteDef (kind:"advance")  • zustand store ติดตาม GPS
   ▼                         │
ปุ่ม "สร้างเส้นทาง · N จุด"   ▼
                      repo.saveDraftRoute(route)  ← เก็บ localStorage "rk.drafts"

๑ หน้า /build — เลือกตัวเลือก

ไฟล์ src/app/build/page.tsx เก็บ state ในเครื่อง 3 อย่าง:

ตัวเลือกชนิดหมายเหตุ
ย่าน (districts)เลือกได้หลายอันมาจาก ALL_DISTRICTS
มรดก (heritage)เลือกได้หลายอันว่าง = เอาทั้งหมดในย่าน
บรรยากาศ (atmosphere)เลือกได้อันเดียวดีฟอลต์ = ATMOSPHERES[1]

มีตัวพรีวิว matchCount ที่คำนวณ "จะได้กี่จุด" ใหม่ทุกครั้งที่เปลี่ยนตัวเลือก เพื่อโชว์บนปุ่ม สร้างเส้นทาง · N จุด · ปุ่มจะกดไม่ได้จนกว่าจะเลือกย่านอย่างน้อย 1 ย่าน

โลจิกคัดจุดถูกเขียนไว้ 2 ที่ — ตัวพรีวิว matchCount ในหน้านี้ กับ pickCheckpoints ในตัว generator ใช้กฎเดียวกัน (รวมถึงกฎ "เหลือ < 2 ให้คงทั้งย่าน") ถ้าแก้กฎ ต้องแก้ทั้งสองที่ ไม่งั้นเลขพรีวิวกับผลจริงจะไม่ตรงกัน

๒ generateAdvanceRoute — ปั้นเส้นทาง

เมื่อกดปุ่ม จะเรียก generateAdvanceRoute(...) ใน src/lib/routeGen.ts ทำ 3 ขั้น:

กคัดหมุด — pickCheckpoints

  • กรอง CHECKPOINTS เอาเฉพาะหมุดในย่านที่เลือก
  • ถ้ามีเลือกมรดก → กรองด้วยมรดกนั้น แต่จะใช้ก็ต่อเมื่อเหลือ ≥ 2 จุด ไม่งั้น fallback กลับไปใช้หมุดทั้งหมดในย่าน (กันเส้นทางสั้นเกิน)

ขเรียงลำดับ — orderByNearestNeighbour

  • เริ่มจากหมุด เหนือสุด (latitude มากสุด)
  • ไล่หยิบหมุดที่ ใกล้จุดล่าสุดที่สุด (ระยะ Haversine) ไปเรื่อย ๆ → ได้เส้นต่อเนื่อง ไม่กระโดดไปมา

คสร้าง RouteDef

{
  id: `advance-${Date.now()}`,   // id ไม่ซ้ำ ติด timestamp
  name: "เส้นทางของฉัน (Advance)",
  atmosphere,                     // บรรยากาศที่เลือก (เป็น metadata)
  distanceKm,                     // จาก pathLengthM(path)
  legCalories: legCaloriesFromPath(path),  // แคลต่อช่วง คิดจากระยะ
  kind: "advance",
  checkpointIds: [...],
  path,                           // เส้นตรงเชื่อมหมุด (placeholder)
}
คืน null ถ้าได้หมุดน้อยกว่า 2 จุด → หน้า /build โชว์ข้อความ "เลือกย่านให้ได้อย่างน้อย 2 จุดเช็คอินก่อนนะครับ"
แคลอรี่: เส้นทาง Advance สร้างสดจึงกรอกแคลเองไม่ได้ → ใช้ legCaloriesFromPath คิดแคล "ต่อช่วง" จากระยะให้อัตโนมัติ (ต่างจาก Basic ที่กรอก legCalories มือได้)

๓ saveDraftRoute — ส่งต่อให้หน้าวิ่ง

เส้นทางที่เพิ่งสร้าง ไม่มีอยู่ใน routes.ts เลยหาด้วย id ปกติไม่ได้ จึง:

  • repo.saveDraftRoute(route) เก็บลง localStorage คีย์ rk.drafts เป็น map { id: RouteDef }
  • เป็น local แบบ sync ที่ทั้งสอง repo (Local/Supabase) ใช้ร่วมกัน — draft เป็นข้อมูลเฉพาะเครื่อง ไม่ sync ขึ้น cloud เพราะเป็นแค่ตัวส่งต่อ build→run
  • แล้ว router.push(/run/<route.id>)

๔ หน้า /run — หยิบ draft + เกาะถนน

ไฟล์ src/app/run/[routeId]/page.tsx resolve เส้นทางแบบนี้:

// หา Basic ก่อน ไม่เจอค่อยดู draft (Advance) ใน localStorage
const base = basicRouteById(routeId) ?? repo.getDraftRoute(routeId);

id ที่ขึ้นต้นด้วย advance- จะไม่เจอใน Basic → ดึง draft กลับมาจาก localStorage จากตรงนี้เส้นทาง custom ทำงานเหมือนเส้นทางสำเร็จรูปทุกอย่าง: path เส้นตรง จะถูก "ดัด" ให้เกาะถนนจริงผ่าน ORS (snapToRoads → /api/route) แล้ว zustand store จัดการ GPS/sim + เช็คอินตามเดิม

Basic vs Advance ต่างกันยังไง

ประเด็นBasicAdvance (custom)
นิยามที่ไหนdata/routes.ts (เขียนไว้ล่วงหน้า)สร้างสดตอนกดปุ่ม
idคงที่advance-<timestamp>
เก็บที่ไหนในโค้ดlocalStorage rk.drafts
แคลอรี่กรอกมือได้คิดจากระยะอัตโนมัติ
ลำดับหมุดกำหนดเองnearest-neighbour
เกาะถนน (ORS)เหมือนกันทั้งคู่

⚠️ ข้อควรระวัง

  • กฎคัดจุดอยู่ 2 ที่ (พรีวิว + generator) — แก้ต้องแก้พร้อมกัน
  • draft เป็น local-only และผูกกับ id — เส้นทาง custom อยู่แค่ในเครื่องที่สร้าง รีเฟรช /run/advance-123... ได้เพราะ draft ยังอยู่ใน localStorage แต่เปิดในอีกเครื่อง จะหาไม่เจอ
  • บรรยากาศ (atmosphere) เป็นแค่ metadata — ไม่มีผลต่อการคัดหมุดหรือการ routing เก็บติดไปกับเส้นทางเพื่อแสดงผลเท่านั้น
